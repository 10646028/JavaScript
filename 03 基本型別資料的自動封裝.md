# 03 基本型別資料的自動封裝

```
在string(基本型別資料)呼叫String(內建物件)的方法時, 基本型別資料先被自動封裝(auto boxing)成物件, 再呼叫該物件的方法.
```

### 程式範例

```javascript
//--------------------------------
// m1是一個基本資料型別(string),
// 不是字串物件;
// 當在m1上呼叫字串物件的方法時,
// m1先被自動封裝成String Object,
// 再呼叫該物件的方法.
//--------------------------------
var m1 = 'abcde';

console.log('m1的型態: ' + typeof(m1));
console.log('m1是字串物件? ' + (m1 instanceof String));
console.log('m1呼叫字串物件的方法: ' + m1.charAt(0));
console.log()

//--------------------------------
// m2是一個String Object,
// 作為與m1的對比;
// 多數時候使用字串類似m1的方式.
//--------------------------------
var m2 = new String('abcde');

console.log('m2的型態: ' + typeof(m2));
console.log('m2是字串物件? ' + (m2 instanceof String));
console.log('m2呼叫字串物件的方法: ' + m2.charAt(0));
console.log()

//--------------------------------
// 字串物件是一個Immutable物件.
//--------------------------------
var m3 = m2;
m2 = m2.toUpperCase();

console.log('新m2的內容: ' + m2);
console.log('m2的型態: ' + typeof(m2));
console.log('m3的型態: ' + typeof(m3));
```
